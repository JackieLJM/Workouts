jTeaser Jquery Plugin What is jTeaser

Have you ever wished to have a nice javascript component to spice up your web site that enables you to highlight some content of your web site with an image and text teaser (small image, a teaser text and link to your HTML content). With jTeaser, that becomes true for you if you cannot get a flash component. This feature is often needed for many sites.

 

 

Features

The jTeaser jquery plugin features :

  • Teaser text with image to spice up your web front page;
  • Loop's once or infinitly or not at all;
  • Multiple jTeaser on the same page;
  • Build on jQuery 1.4.2;
jTeaser de-mystified The component

The component is devised into three sub-components:

  • The image viewer : the view to yours images;
  • The buttons: the buttons which can display custom HTML, or a title & a subtitle as text;
  • The magnifier: highlight content of one button.
The image viewer

For each button, you must provide one image. The image is displayed when you click on a button.

The buttons

The buttons are rectangle shaped and contains some HTML content, or a default title and text string. The buttons are always placed on the right side of the component.

The magnifier

The buttons are highlighted with a magnifier, which can either display alternate content for the button, or display's the same content, but magnified. You can click on the magnifier, or the image, to access to the specified content using url's.

Calculating the total size of the component

As you don't want to rely on the default size, it is possible for you to adjust the dimension for this component. To do so, you will need to set one or all of theses elements:

  • nbButtons : the number of buttons you want displayed in the component;
  • buttonWidth : the button width in pixels of the buttons;
  • buttonHeight : the height in pixels of the buttons;
  • imageWidth : the width in pixels of the images.

Thus, you can set the width and height of the component by providing your own layout settings for nbButtons, buttonWidth, buttonHeight, imageWidth.

Javascript inclusions

To make jTeaser work, you must include these HTML markups:

<script language="JavaScript" type="text/javascript" src="js/jquery/jquery-1.4.2.js"></script>
<script language="JavaScript" type="text/javascript" src="js/jquery.jTeaser.min.js"></script>

The component html markup
<div id="teaser0"></div>
or
<div id="teaser0"/>

The component teaser data - the javascript way

One must initialize the component with data, called teaser data. There is also an alternative way to provide the data using HTML, called HTML markup teaser data. The teaser data consists of an array of objects.

  • The teaser data object consists of theses properties :
    • title [mandatory]: The title for both the button and magnifier.
    • text [mandatory]: The text for both the button and magnifier.
    • textMagnifier [optional]: If provided, the text for the magnifier, overriding the text property;
    • image [mandatory]: The image url;
    • url [mandatory]: The url to a html content;

You will need one teaser data for each teaser. The array must be of the same length of nbButtons, which is by default of 4; the component will display 4 teasers. Thus, in this case, you must provide an array of teaser data object of length 4.

The component initialisation upon instantiation

Here is the javascript default options that is passed to jTeaser() and that you can modify to your convinience.

{
    debug: false, // The debug fag for this component
    mode: 'display', // [display|slide]
    orientation: 'horizontal', // if (mode=='slide') [vertical|horizontal]
    teasers: null, // The teasers data
    layout: {
        nbButtons: 4, // the number of buttons
        buttonWidth: 150, // the width of the buttons in pixels
        buttonHeight: 63, // the height of the buttons in pixels
        imageWidth: 350 // The width of the image in pixels
    },
    loop: 'once', // [none|once|infinite]loop the magnifier once or infinitly
    loopPause: 5000, // while looping, the magnifier pause for each button
    showText: true, // shows text in buttons, if not, only the titles are displayed
    intro: true, // slide the magnifier from left to right as introduction
    message: null,
    style: { // auto explanatory
        border : '1px solid silver', // css border for the component
        separatorColor: '#FFFFFF', // separator color between buttons for the component

The component teaser data - the HTML markup way

If the teaser data is set to null, the component will look for HTML markup teaser data within the targeted element. With this technique, you have full control of what is shown since you provide HTML content for both the buttons and the magnifier, instead of using the default behaviour. Thus, you wont have to provide a style object option to the component. In this case, you must set the teasers option to null and the style is simply overridden.

With this technique, you thus setup teaser data with the HTML markup, and also, you control the layout & the styling with the HTML markup, and this for both the buttons and the magnifier.

<div id="teaser5" style="display: none; ">
  <div id="teasers">
    <div id="teaser1" data-image="" data-url="">...</div>
    <div id="teaser2" data-image="" data-url="">...</div>
    <div id="teaser3" data-image="" data-url="">...</div>
    <div id="teaser4" data-image="" data-url="">...</div>
    <div id="teaser5" data-image="" data-url="">...</div>
    <div id="magnifier1">...</div>
    <div id="magnifier2">...</div>
    <div id="magnifier3">...</div>
    <div id="magnifier4">...</div>
    <div id="magnifier5">...</div>
  </div>
</div>
